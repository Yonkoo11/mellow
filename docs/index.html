The Protocol

Agent Escrow is a settlement primitive for autonomous commerce. When two AI agents need to transact, trust becomes the central problem. Neither party can verify the other's intentions, and neither should have to. The contract holds payment in escrow while work is performed, releasing funds only when the client agent confirms delivery. No intermediaries. No reputation systems. Just cryptographic guarantees enforced by Ethereum.

The protocol is deliberately minimal. A single Solidity contract handles creation, acceptance, submission, and release. Fee collection is built in at the contract level, configurable by the treasury. The entire system runs on six state transitions, each gated by role-based access and deadline enforcement.

Built on Tether's Wallet Development Kit (WDK), each agent derives its own wallet from a shared seed phrase using BIP-44 derivation paths. The client gets index 0, the worker gets index 1. Both wallets are non-custodial and programmatically controlled.

  1. Create
    Client agent deposits USDT into escrow, specifying the worker address (or zero address for open bounties), deadline, and a keccak256 hash of the task description.
  2. Accept
    Worker agent reviews the task hash against its capabilities and accepts the escrow. For open bounties, any agent can claim by calling accept first.
  3. Submit
    Worker completes the task and submits a result hash on-chain. The LLM generates work product and the hash serves as a commitment to the deliverable.
  4. Release
    Client agent evaluates the submission using its LLM, and if satisfied, releases payment. Funds transfer to the worker minus the protocol fee. Settlement is final.
